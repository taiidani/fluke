# Agents

Fluke's documentation is the source of truth for architecture, design decisions, and conventions — for both humans and AI coding agents.

Before contributing, read:

1. [Architecture](docs/explanation/architecture.md) — system components, data flow, design constraints
2. [Executor model](docs/explanation/executor-model.md) — the core extension point; understand this before touching executor code
3. [Statelessness](docs/explanation/statelessness.md) — what state exists, where, and why
4. [Security model](docs/explanation/security-model.md) — threat model and trust boundaries
5. [Configuration reference](docs/reference/configuration.md) — HCL schemas for server and agent
6. [CONTRIBUTING.md](CONTRIBUTING.md) — build setup, project structure, PR conventions

When in doubt about a design decision, check the explanation docs before assuming.
